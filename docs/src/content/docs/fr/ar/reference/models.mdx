L'action GitHub `pelikhan/action-continuous-translation` prend en charge plusieurs fournisseurs de LLM différents via [GenAIScript](https://microsoft.github.io/genaiscript/configuration/).

## Choisir les modèles

Vous pouvez fournir un schéma complet qui associe une langue à un identifiant de modèle pour contrôler le modèle utilisé pour chaque langue via le paramètre `model_alias`. Pour chaque langue, vous pouvez définir deux modèles, un pour la `translation` et un autre pour la `validation`, sous la forme d'un objet YAML.

```yaml
with:
  model_alias: |
    translation_fr: github:openai/gpt-4.1
    validation_es: openai:gpt-4o
```

## Configurer les secrets

Vous devez passer explicitement les secrets du fournisseur LLM sous forme de paramètres à l'action.

```yaml
with:
  github_token: ${{ secrets.GITHUB_TOKEN }}
  openai_api_key: ${{ secrets.OPENAI_API_KEY }}
  openai_api_base: ${{ env.OPENAI_API_BASE }}
  azure_openai_api_endpoint: ${{ env.AZURE_OPENAI_API_ENDPOINT }}
  azure_openai_api_key: ${{ secrets.AZURE_OPENAI_API_KEY }}
  azure_openai_subscription_id: ${{ env.AZURE_OPENAI_SUBSCRIPTION_ID }}
  azure_openai_api_version: ${{ env.AZURE_OPENAI_API_VERSION }}
  azure_openai_api_credentials: ${{ env.AZURE_OPENAI_API_CREDENTIALS }}
```

### `github_token`

**Type** : `string` (de préférence un secret)\
**Valeur par défaut** : `${{ secrets.GITHUB_TOKEN }}`

Votre [GitHub token](https://github.com/settings/tokens) avec au moins les permissions `models: read`. Consultez plus de détails dans la [documentation des permissions des modèles GitHub dans GenAIScript](https://microsoft.github.io/genaiscript/reference/github-actions/#github-models-permissions).

### `openai_api_key`

**Type** : `string` (de préférence un secret)\
**Valeur par défaut** : `${{ secrets.OPENAI_API_KEY }}`

La clé d'API OpenAI. En savoir plus [ici](https://help.openai.com/en/articles/4936850-where-do-i-find-my-openai-api-key).

### `openai_api_base`

**Type** : `string` (de préférence une variable d'environnement)\
**Valeur par défaut** : `${{ env.OPENAI_API_BASE }}`

L'URL de base de l'API d'OpenAI.

### `azure_openai_api_endpoint`

**Type** : `string` (de préférence une variable d'environnement)\
**Valeur par défaut** : `${{ env.AZURE_OPENAI_API_ENDPOINT }}`

Point de terminaison Azure OpenAI. Pour le trouver, accédez à votre portail Azure, ouvrez votre ressource Azure OpenAI, recherchez les clés et points de terminaison, puis copiez le point de terminaison.

### `azure_openai_api_key`

**Type** : `string` (de préférence un secret)\
**Valeur par défaut** : `${{ secrets.AZURE_OPENAI_API_KEY }}`

La clé d'API Azure OpenAI.

:::note
Vous n'en avez pas besoin si vous utilisez Microsoft Entra ID.
:::

### `azure_openai_subscription_id`

**Type** : `string` (de préférence une variable d'environnement)\
**Valeur par défaut** : `${{ env.AZURE_OPENAI_SUBSCRIPTION_ID }}`

L'identifiant de souscription Azure OpenAI permettant d'afficher les déploiements disponibles.

:::note
Vous en avez besoin uniquement si vous utilisez Microsoft Entra ID.
:::

### `azure_openai_api_version`

**Type** : `string` (de préférence une variable d'environnement)\
**Valeur par défaut** : `${{ env.AZURE_OPENAI_API_VERSION }}`

La version de l'API Azure OpenAI.

### `azure_openai_api_credentials`

**Type** : `string` (de préférence une variable d'environnement)\
**Valeur par défaut** : `${{ env.AZURE_OPENAI_API_CREDENTIALS }}`

Le type d'identifiant pour Azure OpenAI. Laissez ce paramètre sur `'default'`, sauf si vous avez une configuration spécifique à Azure.
