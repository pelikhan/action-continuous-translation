Vous pouvez insérer des directives personnalisées pour guider le processus de traduction. Cela peut être utilisé pour ajouter des instructions supplémentaires, un glossaire ou d'autres orientations afin d'améliorer la qualité globale de la traduction.

## Directives intégrées

```yaml title=".github/workflows/ct.yml" "instructions: Use a casual tone."
uses: pelikhan/action-continuous-translation@v0
with:
  instructions: Use a casual tone.
```

## Directives à partir d'un fichier

```yaml title=".github/workflows/ct.yml" "instructions_file: ./instructions.txt"
uses: pelikhan/action-continuous-translation@v0
with:
  instructions_file: ./instructions.txt
```

## Directives dans les métadonnées

Vous pouvez également spécifier des directives de traduction directement dans les métadonnées de documents individuels. Cela est utile lorsque des documents différents nécessitent une approche de traduction spécifique.

```markdown title="example.md"
---
title: "My Document"
translator:
  instructions: |
    Use formal tone and preserve all technical terms in English.
---

# My Document Content

This document will be translated using the instructions specified in the frontmatter.
```

Les types `translator` sont définis dans `types.ts`. Ils sont supprimés des métadonnées avant le début de la traduction, donc ils n'affectent pas le résultat final.

## Ordre de Priorité

Les directives sont appliquées selon l'ordre de priorité suivant :

1. Paramètre `instructions` (la priorité la plus élevée)
2. Paramètre `instructions_file`
3. `translator.instructions` des métadonnées (la priorité la plus basse)

Si plusieurs méthodes sont utilisées, seule celle avec la priorité la plus élevée sera appliquée.
