A ação do GitHub `pelikhan/action-continuous-translation` oferece suporte a vários provedores de LLM diferentes por meio do [GenAIScript](https://microsoft.github.io/genaiscript/configuration/).

## Escolha de Modelos

Você pode fornecer um mapeamento completo que associa o idioma ao identificador do modelo para controlar o modelo usado para cada idioma através do parâmetro `model_alias`.
Para cada idioma, você pode especificar dois modelos, um para `translation` e outro para `validation`, como um objeto YAML.

```yaml
with:
  model_alias: |
    translation_fr: github:openai/gpt-4.1
    validation_es: openai:gpt-4o
```

## Configuração de Segredos

Os segredos do provedor de LLM devem ser passados explicitamente como parâmetros para a ação.

```yaml
with:
  github_token: ${{ secrets.GITHUB_TOKEN }}
  openai_api_key: ${{ secrets.OPENAI_API_KEY }}
  openai_api_base: ${{ env.OPENAI_API_BASE }}
  azure_openai_api_endpoint: ${{ env.AZURE_OPENAI_API_ENDPOINT }}
  azure_openai_api_key: ${{ secrets.AZURE_OPENAI_API_KEY }}
  azure_openai_subscription_id: ${{ env.AZURE_OPENAI_SUBSCRIPTION_ID }}
  azure_openai_api_version: ${{ env.AZURE_OPENAI_API_VERSION }}
  azure_openai_api_credentials: ${{ env.AZURE_OPENAI_API_CREDENTIALS }}
```

### `github_token`

**Tipo**: `string` (preferencialmente segredo)\
**Valor padrão**: `${{ secrets.GITHUB_TOKEN }}`

Seu [GitHub token](https://github.com/settings/tokens) com permissão mínima de `models: read`.
Leia mais detalhes na [documentação de permissões de modelos do GitHub no GenAIScript](https://microsoft.github.io/genaiscript/reference/github-actions/#github-models-permissions).

### `openai_api_key`

**Tipo**: `string` (preferencialmente segredo)\
**Valor padrão**: `${{ secrets.OPENAI_API_KEY }}`

Sua chave de API do OpenAI.
Leia mais [aqui](https://help.openai.com/en/articles/4936850-where-do-i-find-my-openai-api-key).

### `openai_api_base`

**Tipo**: `string` (preferencialmente variável de ambiente)\
**Valor padrão**: `${{ env.OPENAI_API_BASE }}`

A URL base para a API do OpenAI.

### `azure_openai_api_endpoint`

**Tipo**: `string` (preferencialmente variável de ambiente)\
**Valor padrão**: `${{ env.AZURE_OPENAI_API_ENDPOINT }}`

Ponto de extremidade do Azure OpenAI.
Para localizar este ponto, acesse o portal do Azure, abra o seu recurso do Azure OpenAI.
Procure por "Chaves e Pontos de Extremidade" e copie o ponto de extremidade.

### `azure_openai_api_key`

**Tipo**: `string` (preferencialmente segredo)\
**Valor padrão**: `${{ secrets.AZURE_OPENAI_API_KEY }}`

Sua chave de API do Azure OpenAI.

:::note
Você não precisa disso se estiver usando o Microsoft Entra ID.
:::

### `azure_openai_subscription_id`

**Tipo**: `string` (preferencialmente variável de ambiente)\
**Valor padrão**: `${{ env.AZURE_OPENAI_SUBSCRIPTION_ID }}`

O ID de assinatura do Azure OpenAI para visualizar as implantações disponíveis.

:::note
Você só precisa disso se estiver usando o Microsoft Entra ID.
:::

### `azure_openai_api_version`

**Tipo**: `string` (preferencialmente variável de ambiente)\
**Valor padrão**: `${{ env.AZURE_OPENAI_API_VERSION }}`

A versão da API do Azure OpenAI.

### `azure_openai_api_credentials`

**Tipo**: `string` (preferencialmente variável de ambiente)\
**Valor padrão**: `${{ env.AZURE_OPENAI_API_CREDENTIALS }}`

O tipo de credenciais do Azure OpenAI.
Deixe como `'default'` a menos que você tenha uma configuração específica para o Azure.
