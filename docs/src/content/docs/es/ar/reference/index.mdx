La acción personalizada utiliza [Modelos de GitHub](https://github.com/models) para traducir archivos Markdown en tu repositorio.
Está diseñada para usarse en un flujo de trabajo de [GitHub Actions](https://docs.github.com/en/actions) para traducir automáticamente los archivos de documentación cuando se actualicen.

```yaml title=".github/workflows/ct.yml"
- uses: pelikhan/action-continuous-translation@v0
  with:
    github_token: ${{ secrets.GITHUB_TOKEN }}
    lang: fr,es
```

## Instalación

Puedes guardar este archivo en tu directorio `.github/workflows/` como `ct.yml`:

```yaml title=".github/workflows/ct.yml" wrap
name: Continuous Translation
on:
  workflow_dispatch:
  # translate when doc files are changed in main
  push:
    branches:
      - main
    paths: # update the file paths
      - "README.md"
      - "docs/src/content/docs/**"
permissions:
  contents: write
  # allow the action to use the GitHub Models API
  models: read
concurrency:
  # cancel in-progress jobs for the same workflow and ref
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true
jobs:
  continuous_translation:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      # cache the LLM inference results
      - uses: actions/cache@v4
        with:
          path: .genaiscript/cache/**
          key: continuous-translation-${{ github.run_id }}
          restore-keys: |
            continuous-translation-
      # this is the translation action
      - uses: pelikhan/action-continuous-translation@v0
        continue-on-error: true # don't stop if translation fails, we still need to store the cache
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          lang: fr,es
      # commit the generated files
      - uses: stefanzweifel/git-auto-commit-action@v5
        with:
          file_pattern: "**.md* translations/**"
          commit_message: "[cai] translated docs"
          commit_user_name: "genaiscript"
```

## Parámetros

La acción `pelikhan/action-continuous-translation` en GitHub admite los siguientes parámetros.
Todos los parámetros son opcionales.
Los valores predeterminados se indican en la referencia a continuación.

```yaml
with:
  lang: fr
  source: en
  files: README.md
  instructions:
  instructions_file:
  glossary_file:
  starlight_dir:
  starlight_base:
  force: false
  model_alias: |
    translate: github:gpt-4o-mini
  debug: false
```

Consulta [Modelos](../../reference/models/) para obtener más detalles sobre otros proveedores como OpenAI, Azure OpenAI, etc.

### `lang`

**Tipo**: `string`\
**Valor predeterminado**: `fr`

Lista de códigos ISO de los idiomas de destino, separados por comas (`,`), a los que se deben traducir los documentos desde el [idioma fuente](#source).
Consulta una lista de todos los idiomas compatibles [en el archivo `models.mts` del repositorio](https://github.com/pelikhan/action-continuous-translation/blob/main/genaisrc/src/models.mts).

### `source`

**Tipo**: `string`\
**Valor predeterminado**: `en`

Código ISO del idioma de origen en el que se ha escrito el contenido principal.

### `files`

**Tipo**: `string`\
**Valor predeterminado**: `README.md`

Lista de archivos que se procesarán, separados por comas.

### `instructions`

**Tipo**: `string`

Instrucciones adicionales para usar el modelo LLM durante la traducción.

### `instructions_file`

**Tipo**: `string`

Ruta a un archivo que contiene instrucciones adicionales para usar el modelo LLM durante la traducción.

### `glossary_file`

**Tipo**: `string`

Ruta a un archivo que contiene un glosario de términos para usar durante la traducción.

### `translations_dir`

**Tipo**: `string`\
**Valor predeterminado**: `translations`

El directorio donde se almacenarán las traducciones.

### `max_translation_tokens`

**Tipo**: `integer`
**Valor predeterminado**: `8000`

Número máximo de tokens que se pueden procesar en una llamada de traducción utilizando el modelo LLM.

### `max_validation_tokens`

**Tipo**: `integer`
**Valor predeterminado**: `2000`

Número máximo de tokens que se pueden procesar en una llamada de validación utilizando el modelo LLM.

### `starlight_dir`

**Tipo**: `string`

El directorio raíz para la documentación de [Astro Starlight](../../reference/astro-starlight/).
Debe configurarse si se define la opción [`starlight_base`](#starlight_base).

### `starlight_base`

**Tipo**: `string`

El alias base para la documentación de [Astro Starlight](../../reference/astro-starlight/).
Si no has configurado la opción [Base de Astro](https://docs.astro.build/en/reference/configuration-reference/#base), no especifiques esta opción.

### `force`

**Tipo**: `boolean`\
**Valor predeterminado**: `false`

Forzar la traducción incluso si el archivo ya ha sido traducido.

### `debug`

**Tipo**: `boolean`\
**Valor predeterminado**: `false`

Habilitar el registro de depuración.
Obtén más detalles en la [documentación de registro de GenAIScript](https://microsoft.github.io/genaiscript/reference/scripts/logging/).

### Alias de modelo

**Tipo**: `string`\
**Ejemplo**: `translate: github:gpt-4o-mini`

Asignación de nombres de modelos a sus alias como un objeto YAML. Esto se puede utilizar para definir el modelo a emplear en traducciones específicas.

### `github_token`

**Tipo**: `string` (se prefiere que sea un secreto)\
**Ejemplo**: `${{ secrets.GITHUB_TOKEN }}`

Tu token de GitHub [GitHub token](https://github.com/settings/tokens) con al menos permiso de `models: read`.

Obtén más detalles en la [documentación de permisos de modelos de GitHub para GenAIScript](https://microsoft.github.io/genaiscript/reference/github-actions/#github-models-permissions).

### Otros proveedores de LLM

También puedes usar otros proveedores de LLM como OpenAI, Azure OpenAI, etc.
Consulta [Modelos](../../reference/models/) para obtener más detalles sobre otros proveedores.
