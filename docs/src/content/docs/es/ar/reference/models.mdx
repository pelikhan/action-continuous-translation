La acción de GitHub `pelikhan/action-continuous-translation` admite varios proveedores de LLM a través de [GenAIScript](https://microsoft.github.io/genaiscript/configuration/).

## Selección de modelos

Puedes proporcionar un esquema completo que vincule el idioma con el identificador del modelo para controlar el modelo utilizado para cada idioma a través del parámetro `model_alias`.
Para cada idioma, puedes especificar dos modelos: uno para `translation` y otro para `validation` como un objeto YAML.

```yaml
with:
  model_alias: |
    translation_fr: github:openai/gpt-4.1
    validation_es: openai:gpt-4o
```

## Configuración de secretos

Debes transferir explícitamente los secretos del proveedor de LLM como parámetros para la acción.

```yaml
with:
  github_token: ${{ secrets.GITHUB_TOKEN }}
  openai_api_key: ${{ secrets.OPENAI_API_KEY }}
  openai_api_base: ${{ env.OPENAI_API_BASE }}
  azure_openai_api_endpoint: ${{ env.AZURE_OPENAI_API_ENDPOINT }}
  azure_openai_api_key: ${{ secrets.AZURE_OPENAI_API_KEY }}
  azure_openai_subscription_id: ${{ env.AZURE_OPENAI_SUBSCRIPTION_ID }}
  azure_openai_api_version: ${{ env.AZURE_OPENAI_API_VERSION }}
  azure_openai_api_credentials: ${{ env.AZURE_OPENAI_API_CREDENTIALS }}
```

### `github_token`

**Tipo**: `string` (preferentemente secreto)\
**Valor predeterminado**: `${{ secrets.GITHUB_TOKEN }}`

Tu [GitHub token](https://github.com/settings/tokens) con permiso de al menos `models: read`.
Consulta más detalles en la [documentación de permisos de modelos de GitHub en GenAIScript](https://microsoft.github.io/genaiscript/reference/github-actions/#github-models-permissions).

### `openai_api_key`

**Tipo**: `string` (preferentemente secreto)\
**Valor predeterminado**: `${{ secrets.OPENAI_API_KEY }}`

Clave de API de OpenAI.
Consulta más información [aquí](https://help.openai.com/en/articles/4936850-where-do-i-find-my-openai-api-key).

### `openai_api_base`

**Tipo**: `string` (preferentemente como variable de entorno)\
**Valor predeterminado**: `${{ env.OPENAI_API_BASE }}`

URL base para la API de OpenAI.

### `azure_openai_api_endpoint`

**Tipo**: `string` (preferentemente como variable de entorno)\
**Valor predeterminado**: `${{ env.AZURE_OPENAI_API_ENDPOINT }}`

Punto de conexión de Azure OpenAI.
Para localizar este punto, ve al portal de Azure, abre tu recurso de Azure OpenAI.
Busca "claves y puntos de conexión" y copia el punto final.

### `azure_openai_api_key`

**Tipo**: `string` (preferentemente secreto)\
**Valor predeterminado**: `${{ secrets.AZURE_OPENAI_API_KEY }}`

Clave de API de Azure OpenAI.

:::note
No necesitas esto si usas Microsoft Entra ID.
:::

### `azure_openai_subscription_id`

**Tipo**: `string` (preferentemente como variable de entorno)\
**Valor predeterminado**: `${{ env.AZURE_OPENAI_SUBSCRIPTION_ID }}`

ID de suscripción de Azure OpenAI para visualizar las implementaciones disponibles.

:::note
Solo necesitas esto si usas Microsoft Entra ID.
:::

### `azure_openai_api_version`

**Tipo**: `string` (preferentemente como variable de entorno)\
**Valor predeterminado**: `${{ env.AZURE_OPENAI_API_VERSION }}`

Versión de la API de Azure OpenAI.

### `azure_openai_api_credentials`

**Tipo**: `string` (preferentemente como variable de entorno)\
**Valor predeterminado**: `${{ env.AZURE_OPENAI_API_CREDENTIALS }}`

Tipo de credencial de Azure OpenAI.
Déjalo como `'default'` a menos que tengas una configuración personalizada en Azure.
