Puedes introducir directrices personalizadas para guiar el proceso de traducción. Esto se puede utilizar para agregar instrucciones adicionales, un glosario u otras orientaciones para mejorar la calidad de la traducción en general.

## Directrices integradas

```yaml title=".github/workflows/ct.yml" "instructions: Use a casual tone."
uses: pelikhan/action-continuous-translation@v0
with:
  instructions: Use a casual tone.
```

## Directrices desde un archivo

```yaml title=".github/workflows/ct.yml" "instructions_file: ./instructions.txt"
uses: pelikhan/action-continuous-translation@v0
with:
  instructions_file: ./instructions.txt
```

## Directrices en el encabezado

También puedes especificar directrices de traducción directamente en el encabezado de documentos individuales. Esto es útil cuando los documentos requieren un enfoque de traducción específico.

```markdown title="example.md"
---
title: "My Document"
translator:
  instructions: |
    Use formal tone and preserve all technical terms in English.
---

# My Document Content

This document will be translated using the instructions specified in the frontmatter.
```

Los tipos de `translator` se definen en `types.ts`. Se eliminan del encabezado antes de iniciar la traducción, por lo que no afectan al resultado final.

## Orden de prioridad

Las directrices se aplican según el siguiente orden de prioridad:

1. El parámetro `instructions` (máxima prioridad)
2. El parámetro `instructions_file`
3. `translator.instructions` del encabezado del documento (menor prioridad)

Si se utilizan varios métodos, solo se aplicará el de mayor prioridad.
